An Error Message! No Reason To Panic!
-------------------------------------
- Syntax errors are immediately highlighted by VS Code
    - Represented as a Red squigly line beneath the captured error 
        - Hovering over provides a description of the problem

    - The Explorer View highlights the file (with error) in Red


- To address logical errors
    - Use the browser's built-in dev tools
        - Preferably switch to Incognito Mode
        - Prevents installed extensions from interferring with debugging

    - Click on the Console tab
        - Display errors NOT detected by VS Code 
        - Provide the number line of code and file associated with the error 
        - Clicking the number line
            - Opens the Source tab
            - Loads the associated file
            - Highlights the errornous line of code
