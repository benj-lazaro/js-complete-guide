Introducing Rest Parameters (Rest Operator)
-------------------------------------------
- The Rest operator (...) allows a function to
    - Accept an indefinite amount of argument values in the form of an array

    - It is similar to that of the Spread operator (...)
        - EXCEPT it takes all argument values & merges them into an array
    
    - The "..." operator prefixes a parameter variable
        
    - The "..." operator MUST be positioned as the last formal parameter
        - Otherwise, triggers an "Uncaught SyntaxError: Rest parameter must be last formal parameter"

    - NOTE: ONLY one (1) Rest parameter is ALLOWED
        - The 1st Rest parameter alone merges ALL argument values together in an array
        - Leaving NO room for a 2nd Rest parameter

    Syntax:
    function <name>(<parameter>, ...<parameter_name>) {
        // statement
    }

    const <name> = (<parameter>, ...<parameter_name>) => {
        // statement
    }


- An alternative to "..." operator
    - But ONLY works on a function declaration that
        - Uses the keyword "function" & w/o a defined parameter variable

    - JavaScript offers an special "invisible" parameter variable named "arguments"
        - This was available BEFORE the "..." operator was introduced in ES6

    - The "arguments" operator like that of the "..."
        - It takes all argument values & merges them into an array-like object

    - NOTE: Does NOT work on Arrow functions due to the absence of the keyword "function"
        - Using the "..." operator is RECOMMENDED over "arguments"
