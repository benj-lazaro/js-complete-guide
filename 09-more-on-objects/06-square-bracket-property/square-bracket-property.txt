Special Key Names & Square Bracket Property Access
--------------------------------------------------
- On naming an object's property
    - Any name that can be used as a variable name
        - HOWEVER, NOT every property name can be used as a variable name

    - A property name is MORE FLEXIBLE than a variable name
    
    - Enclosing the property name w/in a pair of " " allows 
        - A "-", whitespace or both placed in-between words of a property name
        
        - NOTE: This is exclusive ONLY to objects

    - A property name is AUTOMATICALLY COERCED as a String data type
        - Assigned value can be of any data type


- To access a special property name
    - Use the Square bracket notation
    - Pass the special property name enclosed w/in a " " as arguement
        
    Syntax:
    <object>.["<property-name>"];
    <object>.["<property name>"];


- The Square bracket notation can also be used to
    - Set a CSS property of a selected Element node
    
    - Using the original CSS property name wrapped w/in a pair of " "
        - Instead of it's camelCase equivalent 


- On the practice project (assets/scripts/object.js)
    - Refactor the property name "name" into "first name"

    - Access the special property name "first name"
        - Using the Square bracket notation

    - Select the HTML element <ul> w/ the id of "movie-list" from the HTML document

            - Assign the value "block"
