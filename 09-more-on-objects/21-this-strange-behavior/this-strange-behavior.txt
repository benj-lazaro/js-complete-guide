The "this" Keyword & Its Strange behavior
-----------------------------------------
- When a method of an object uses the keyword "this" w/in own its body
    - The keyword "this" refers to the object that called the method
    - e.g. <object_name>.<method_name>();


- When the same method is extracted from an object & stored in a global variable
    - Calls the method as a stand-alone function w/o declaring its associated object

    - On non-strict mode
        - The keyword "this" refers to the global Window object

    - On strict mode
        - The keyword "this" returns the value "undefined"


- To prevent help identify which object that the keyword "this" is currently referreing to
    - Declare the name of the object that calls the method
        - Prefixing the object's name
        - A "." notation
        - Then the method name w/ a pair of "( )"

        Syntax:
        <object_name>.<method_name>();


    - On the variable that stores the extracted object's method
        - Chain the method ".bind()"
            - Pass the intended object to be associated w/ the keyword "this" as argument

        - This explicitly identifies the object that
            - The keyword "this" will refer to
            - When calling the extracted method 

        Syntax:
        let {<extracted_method>} = <object>;
        <variable> = <extracted_method>.bind(<object>);


    - NOTE: BOTH approaches reduces the chance of the keyword "this"
        - Referring to the global Window object
        - Returning the value "undefined
