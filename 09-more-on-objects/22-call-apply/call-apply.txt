.call() And .apply()
--------------------
- The method ".bind()"
    - Returns a pre-configure function object that will be executed later

    - If a function that needs to change the value that the keyword "this" is referring to
        - Executes that function right way & does NOT return a new function object

        - The methods ".apply()" or ".call()" are better suited for this task than ".bind()"


- The method ".call()" 
    - It does NOT return a new function object

    - It immediately executes the function object it is attached to

    - It takes multiple arguments
        - 1st           = The object that the keyword "this" references to
        - Succeeding    = Infinite values that the function will work on (optional)
        
        - NOTE: Argument values are individually separated by a ","

    - Reference (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Function/call)


    Syntax:
    <function>.call(<this_object>, <arg_1>, ..., <arg_n>);

    
- The method ".apply()"
    - It does NOT return a new function object

    - It immediately executes the function object it is attached to

    - It takes multiple arguments
        - 1st           = The object that the keyword "this" references to
        - Succeeding    = Values in stored in an array that the function will work on (optional)   

    - Reference (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Function/apply)

    Syntax:
    <function>.call(<this_object>, [<arg_1>, ..., <arg_n>]);
