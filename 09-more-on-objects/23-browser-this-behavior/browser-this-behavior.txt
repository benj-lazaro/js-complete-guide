What The Browser (Sometimes) Does To "this"
-------------------------------------------
- A use case where the object in front of a function object will NOT work
    - Using the method ".bind()" on an event listener

    - The object in front of a function object ONLY works
        - If it is executing the function object on its own (i.e. NOT by an event)


- Another use case where the object in front of a function object will NOT work
    - Using the method ".bind()" on an event listener

    - An event listener using an Anonymous callback function
        - Browser binds the keyword "this" to the DOM Element node that triggered the event

    - NOTE: An event listener using an Arrow callback function
        - Browser binds the keyword "this" to the global Window object


- In the practice project (assets/scripts/objects.js)
    - On the callback function stored in the global constant "searchMovieHandler"
        - Change it from an Arrow function into an Anonymous function

    - Save the changes

    - Load / reload the Browser
        - Launch the browser's JavaScript console

        - Click the button "Search"
        
        - On the JavaScript console 
            - The browser refers the value of the keyword "this"
            - To the selected Element node that is stored in the global constant "searchBtn"
            - (i.e. HTML element <button> w/ the "id" of "search-bth")
