"this" And The Arrow Functions
------------------------------
- In continuation of the previous lecture
    - An event listener using an Arrow callback function
        - Browser binds the keyword "this" to the global Window object
        - Instead of the DOM Element node that triggered the event


    - An advantage of an Arrow function as a callback function
        - It does NOT bind the keyword "this" to ANY object

        - It has a SPECIAL BEHAVIOR
            - Perceiving to be executed outside its own function body
            - Referencing the keyword "this" to the global Window object


    - Every callback function (Anonymous & normal) 
        - Created w/ the keyword "function" has its own keyword "this" binding
        
        - Binds the keyword "this" to the object responsible
            - For the function object's (i.e. method) execution
        

- In the example code (script.js)
    - Define a new object using Object literal notation
        - Save the object in the global constant named "members"

    - Define the following properties of the object
        - teamName          = string
        - people            = an array of string elements
        - getTeamMembers    = a method 

    - The method "getTeamMembers"
        - Renders the individual value of the property "people" w/ the value of "teamName"

    - On the method "getTeamMembers"
        - Iterate through the property "people" using the method ".forEach()"
            - Prefix the keyword "this" to the property "people
            - The keyword "this" references the object "members"

        - Pass an Arrow function as argument to the method ".forEach()"
            - The Arrow function has a parameter variable named "person"

        - On the Arrow function's body
            - Pass the keyword "this" as argument to "console.log()"

            - Next, pass a string that concatenates the values of
                - The object's properties "people" & "teamName"
                
            - Prefix the keyword "this" to the property "teamName'
                - The keyword "this" references the object "members"
            
    - Outside the object "members"
        - Call the method ".getTeamMembers()"
        
        - Browser references the keyword "this" to object "members"
            - Render the individual names w/ the team name 

        - The Arrow function does NOT bind the keyword "this" to ANY object
            - Preceives that it is execute outside of its own function body

            - It "executes" w/in the method "getTeamMembers"
                - Where the keyword "this" references the object "members"


    - Define a new object using Object literal notation
        - Save the object in the global constant named "members2"
            - Copy & paste the properties & method of object "members"

        - On the method "getTeamMembers"
            - Swap the Arrow function w/ an Anonymous function

        - Leave everything as it is

    - Outside of the object "members2"
        - Execute the method "getTeamMembers()" of the object "members"

        - Renders the individual values of the property "people" 
            - BUT returns the value of "undefined" for the property "teamName"

    - The reason behind the "undefined" value for the property "teamName"
        - The keyword "function" used by the Anonymous function
            - Binds the keyword "this" to the object that executes the Anonymous function

        - The Anonymous function is triggered by the method ".forEach()"
            - It is NOT triggered by the object "member2"

            - Since the method ".forEach()" is NOT a method of the object "member2"
                - JavaScript binds the keyword "this" to the global Window object
            
        - The global Window object does NOT have a property named "teamName"
            - This leads to returning the value "undefined"
