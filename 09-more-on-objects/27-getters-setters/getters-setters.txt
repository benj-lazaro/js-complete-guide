Getters & Setters
-----------------
- These are JavaScript features that controls how
    - To set or assign a value to an object's property (i.e Setter)
    - To retrieve an assigned value from an object's property (i.e. Getter)

    - BOTH works like an object's method BUT 
        - Called (or accessed) like an object's property
        - Accessed w/out the pair of "( )" at the end

    - BOTH are NOT needed to be defined for EVERY property of an object
        - One can be defined w/out the other

    - Omitting a Setter from a property
        - Will result to a READ-ONLY property

    - Common use case:
        - To create READ-ONLY properties
        - Add extra (input) validtion for a property w/ a fallback value
        - Setup default transformations to a property's value


- To define a Getter w/in an object
    - Use the keyword "get"
        - Followed by a method name (of your choice)
        - Next a pair of "( )"
        - Then a pair of "{ }" that contains the Getter's logic

        - NOTE: This will be accessed like a property, NOT a method
            - Do NOT use a pair of "( )" when calling the Getter

    - Inside the Getter
        - Use the keyword "return"
            - Next is the object that called the Getter
                - Referenced by the keyword "this"
            
            - Followed by the "internal" property name defined by a Setter
            
        - NOTE: The internal property has a prefix "_" 
            - Followed by the name of an existing property
            - Whose value was being retrieved from

    Syntax:
    <object> = {
        ...,
        get <method_name>() {
            return this._<internal_property>;
        },
        ...
    };


- To define a Setter w/in an object
    - Use the keyword "set"
        - Followed by a method name (of your choice)
        - Next a pair of "( )"
        - Then a pair of "{ }" that contains the Getter's logic

    - On the paraemter field enclosed w/in a pair of "( )"
        - Define the parameter variable(s) whose value will be assigned
            - To an existing property of the object

    - Inside the Setter
        - Perform validation checks on the value of the paraemter variable(s)

        - Define the object that called the Setter
            - Referenced by the keyword "this"

        - Followed by an "internal" property name
            - Prefix w/ a "_" character w/c indicates it is an "internal" property
            - Then the existing property name of the object
            - Next is the "=" operator
            - And assign the value(s) of the paraemter variable(s)     

    Syntax:
    <object> = {
        ...,
        set <method_name>(<args>) { 
            // Validation check logic w/ default value assignment here
            ...
            this._<property_name> = <arg>;
        },
        ...
    };
