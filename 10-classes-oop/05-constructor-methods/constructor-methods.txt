Working With Constructor Methods
--------------------------------
- The Constructor method of a Class
    - Allows the creation of a Class-based object that contains
        - Different values but retain the same structure (i.e. properties)
    
    - Accepts values passed as arguments that are
        - Assigned as initial values to the corresponding Class fields
        
        - IMPORTANT: A Class field w/in a Constructor method is 
            - Now referred as a Class property

        - Remove Class fields from the Class that are used w/in the Constructor method
            - As they will be overwritten by their Class property counterpart

        - Leave behind those that are NOT initialized w/in the Constructor method

    - JavaScript automatically executes a Constructor method during
        - The instantiation of its Class into an object

    - Prevents the occurrences of mistyping or forgetting
        - A Class property w/c can happen when using Object literal notation


- To create a Constructor method in a Class
    - Use the Keyword "constructor" 
        - Followed by a pair of "( )"
        - Where the parameter variables are defined

    - Then by a pair of "{ }" that holds
        - The body of the Constructor method
    
    - NOTE: Do NOT terminate the Constructor method w/ a ";"

    - Inside the Constructor method's body
        - Define a Class property
            - Prefixed w/ the keyword "this"

        - Followed by a "=" operator

        - Next assign the argument variable whose value will be used
            - To initialize the correponding Class property

        - Terminate each statement (of the property) w/ a ";"
        
        - NOTE: Keyword "this" refers to the Class of the Constructor method

    Syntax:
    class <class_name> {
        <field_name> = <default_value>;
        ...
        <field_name>;

        constructor(<parameter_variable>, ...) {
            this.<property_name> = <parameter_variable>;
            ...
            this.<property_name> = <parameter_variable>;
        }
    }


- To call the Constructor method of a Class
    - Use the keyword "new"
        - Followed by the Class name (where the Constructor method resides)
    
    - Then a pair of "( )"
        - Pass values as arguments
    
    - Terminate the statement w/ a ";"

    Syntax:
    new <class_name>(<argument_value>, ...);

            - title

            - description 

        - Inside the Constructor method
            - Assign the argument values to the correponding Class fields
            - Prefix each Class property w/ the keyword "this"
        

    - Switch to the object that is stored in the global constant "productList"
        - On the property "products"
            - Replace hard-coded object w/ a "Product" object

            - Pass the (previous) object's value as arguments
                - To trigger the Constructor method of the Class "Product"

    
    - Save the chanages
        - Load / reload the browser

        - The behavior of the web app remains unchanged
        
        - The hard-coded objects w/in the property "products" are
            - Replaced w/ Class-based objects (i.e. instances) of the Class "Product"
            
        - This ensures the consistency of each object's structure (i.e. properties)
            - Despite the assigment of different values to the correponding properties
