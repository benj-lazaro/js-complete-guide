Fields VS Properties
--------------------
- A Class field
    - Located w/in a Class declaration

    - It defines the properties that an object will contain
        - After a Class is instantiated into an object (i.e. object creation process)

    - In terms of visibility
        - By default, it is "public"
        - Referred as a public Class field

    - NOTE: Defining Class field(s) can be SKIPPED
        - Use the Constructor method to
            - Define a Class field (NOW referred as Class property)
            - Assign the corresponding passed argument value

    - The Class fields are for fields that 
        - Are NOT INITIALIZED w/in the Class' Constructor method


- A Class property
    - Located w/in the Constructor method of a Class

    - Initializes properties w/ values that
        - Are passed as arguments to the Constructor method

    - In terms of visibility
        - By default, it is "public"
        - Referred as a public Class property

    - Values of Class fields of the same name
        - Will be overwritten by the assigned values w/in the Constructor method

    - NOTE: A new Class property can also be defined
        - For the 1st time w/in the Constructor method


- The different between a Class field & property is simply theoretical
    - A Class field becomes a property
        - Due to the Constructor method being called during instantiaion
        - Instantiating a Class into an object
    

- On the practice project (assets/scripts/app.js)
    - Remove the fields of the Class "Product"

    - Save the changes

    - Load / reload the browser
        - The behavior of the web app remains unchanged
