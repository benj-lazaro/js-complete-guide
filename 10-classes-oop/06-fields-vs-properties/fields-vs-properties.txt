Fields VS Properties
--------------------
- A Class field
    - Located inside a Class declaration

    - Defines the properties of a Class-based object
        - After a Class instantiates to an object

    - In terms of visibility
        - It is "public" by default
        - Referred as a public Class field

    - Defining a Class field can be SKIPPED
        - Use a Constructor method to define a Class property
        - Assign the corresponding value of a parameter variable

    - Class fields are foe those (fields) that are
        - NOT INITIALIZED w/in the Constructor method of a Class


- A Class property
    - Located inside the Constructor method of a Class

    - Defines & initializes the properties of a Class-based object
        - With the values from the corresponding parameter variables

    - In terms of visibility
        - It is "public" by default
        - Referred as a public Class property        

    - The values assign to a Class field will be overwritten
        - By a Class property (of the same name) w/in the Constructor method

    - IMPORTANT: A Class property can be defined for the 1st time
        - Inside a Constructor method


- The difference between Class field & Class property is simply theoretical
    - A Class field becomes a Class property 
        - When the Constructor method of a Class is called (triggered)
        - During the instantiaion of a Class into a Class-based object


- On the practice project (assets/scripts/app.js)
    - Simply remove the defined Class fields from the Class "Product"
        - They have been defined & initialized w/in the Constructor method

    - Save the changes

    - Load / reload the browser
        - NO changes on the web app's behavior
