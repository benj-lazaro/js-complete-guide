Using & Connecting Multiple Classes
-----------------------------------
- A Class can also create an object w/call
    - Contains logic that can be split into multiple Classes
        - Then get these Classes in some clever way

    - NOTE: The order of the Classes does NOT MATTER
        - JavaScript parses through the entire codebase BEFORE execution

    - IMPORTANT: A Class MUST be DEFINED FIRST
        - BEFORE instantiated into a Class-based object 


- To define a method of a Class
    - Declare the name of the method
        - Followed by a pari of "()"
            - Define the parameter variable(s) here (if there is/are any)

        - Next is a pair of "{ }"
            - Contain the body of the method

        - Just like w/ defining a Constructor method
            - Do NOT terminate the method w/ a ";"

    Syntax:
    class <class_name> {
        <field_name> = <default_value>;
        ...
        <field_name>;

        constructor(<parameter_variable>, ...) {
            this.<property_name> = <parameter_variable>;
            ...
            this.<property_name> = <parameter_variable>;
        }

        <method_name>() {
            ...
        }        
    }


- To call a method of a Class
    - Instantiate the Class into an object

    - Store the Class-based object in a global or local constant

    - Chain the method name to the constant
        - Followed by a pair of "()"
            - Pass argument value(s)

        - Terminate the statement w/ a ";"
    
    Example:
    const <object_name> = new <class_name>();

    <object_name>.<method_name>();


- On the practice project (assets/scripts/app.js)
    - Create a new Class named "ProductList"
        - This Class stores an array of "Product" objects

    - Inside the Class "ProductList"
        - Define a Class field named "products"
        - Assign an empty array as value

    - Switch to the object stored in the global constant "productList"
        - Cut the objects stored as array elements in the property "products"
        - Leaving an empty array behind

    - Switch back to the Class "ProductList"
        - Paste the array elements as value to the Class field "products"

    - Create a Constructor method named "render()" w/in the Class "ProductList"
        - Leave it empty as the Class field "products" had already been initialiazed

    - Switch back to the object stored in the global constant "productList"
        - On its method "render()"
            - Select & cut the logic
            - Leaving an empty method behind

    - Switch back to the Class "ProductList"
        - On the method "render()"
            - Paste the logic taken from the global constant "productList"

    - Delete the global constant "productList" from the codebase

    - Instantiate the Class "ProductList" into a Class-based object
        - Store the created object into a global constant named "productList"
        - Call the method "render()" of the Class "ProductList"

    - Save the chanages
        - Load / reload the browser
        - The behavior of the web app remains unchanged
