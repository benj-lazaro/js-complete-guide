Using & Connecting Multiple Classes
-----------------------------------
- A Class can also create an object that contains logic that 
    - Can be split into multiple Classes & have them communicate in some clever way

    - NOTE: The order of the Clases w/in the codebase does NOT MATTER
        - JavaScript parses through the codebase PRIOR to execution

    - IMPORTANT: A Class must be DEFINED FIRST
        - Before it is instantiated into a Class-based object 


- To create a method w/in a Class
    - Declare the name of the method
        - Then a pair of "( )" where parameter variables are defined
        - Followed by a pair of "{ }" to hold the logic
        - NOTE: Do NOT terminate the statement w/ a ";"


    Syntax:
    class <class_name> {
        <field_name> = <default_value>;
        ...
        <field_name>;

        constructor(<parameter_variable>, ...) {
            this.<property_name> = <parameter_variable>;
            ...
            this.<property_name> = <parameter_variable>;
        }

        <method_name>() {
            ...
        }        
    }


- To call a method of a Class
    - Instantiate a Class into an object
        - Store the Class-based object in a global or local constant

    - Chain the method name to the (global or local) constant
        - Followed by a pair of "( )" where argument value(s) is/are passed
        - Termiante the statement w/ a ";"
    
    Example:
    const <object_name> = new <class_name>();

    <object_name>.<method_name>();


- On the practice project (assets/scripts/app.js)
    - Create a new Class named "ProductList"
        - This Class simply stores an array of "Product" objects
    
    
    - Inside the Class "ProductList"
        - Define a Class field named "products" & initialize it w/ an empty array
    

    - On the global constant "productList"
        - Cut the array elements of "Product" objects from the property "products"
            - Leaving an empty array behind


    - Back on the Class "ProductList"
        - Paste the "Product" objects as array elements to the Class field "products"

        - Create a Constructor method
            - Leave it empty as the Class field "products" has already been initialiazed


    - Back on the global constant "prodList"
        - Cut the logic of the method "render()"
            - Leaving an empty method behind

        - Delete the constant "productList" from the codebase
            - Including the call for its method "render()"


    - Back (again) on the Class "ProductList"
        - Create an method named "render()"
        - Paste the logic taken from the global constant "productList"


    - At the last line of the codebase
        - Instantiate the Class "ProductList" into a Class-based object
            - Store the created object in a global constant named "productList"

        - Call the method "render()" of the Class "ProductList"

    
    - Save the chanages
        - Load / reload the browser
        - The behavior of the web app remains unchanged


    - Create a new Class named "ProductItem"
        - This Class handles the HTML markup & unique data of each "Product" object read
    
    
    - Inside the Class "ProductItem"
        - Create a Constructor method
            - Define a parameter variable named "product"

            - This will contain the current "Product" object forwarded by
                - The field "product" of the Class "ProductList"


    - Inside the Constructor method
        - Define a Class property named "product"
            - Prefixed w/ the keyword "this"

        - Assign it w/ the parameter variable "product"


    - Create a method named "render()"
        - Leave it empty for the moment 


    - On the method "render()" of the Class "ProductList"
        - Cut the entire "for..of" loop statement 

        - Leaving behind the following statement 

            prodList.append(prodEl);


    - On the method "render()" of the Class "ProductItem"
        - Paste the code taken from the method "render()" of the Class "ProductList"

        - On the property ".innerHTML" of the local constant "prodEl"
            - Change references to local constant "prop" to "this.product"

        - Return the Element node "li" stored in the local constant "prodEl"
            - Using a "return" statement


    - Back on the "for..of" loop of the Class "ProductList" method "render()"
        - Instantiate the Class "ProductItem" into a Class-based object
            - Pass the "for..of" loop's local constant "prod" as argument 
            - NOTE: The local constant "prod" contain the current "Product" object read
        
        - Store the created object in the local constant named "productItem"

        - Next, call the method "render()" of the Class "ProductItem"
            - Chain the method ".render()" to the local constant "productItem"
            
        - Store the returned Element node "li" in the local constant "prodEl"

        
    - Save the chanages
        - Load / reload the browser
        - The behavior of the web app remains unchanged
