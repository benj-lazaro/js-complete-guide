Binding Class Methods & Working With "this"
-------------------------------------------
- For this lesson
    - Create the callback method for the "click" event of the
        - HTML element button w/ the content "Add to Cart"

        - When clicked, renders the following on the browser's JavaScript console
            - Message "Adding to cart"
            - "ProductItem" object details


- On the practice project (assets/scripts/app.js)
    - In the method "render()" of the Class "ProductItem"
        - Before the last line of code


    - Select the HTML element <button> from the HTML markup in the local constant "prodEl"
        - Chain the method ".querySelector()" to the local constant "prodEl"
        - Pass "button" as argument
        - Store the returned Element node "button" in the local constant "addCartButton"


    - On the local constant "addCartButton"
        - Chain the method ".addEventListener()"
        
        - Pass the arguments
            1st = "click"
            2nd = "this.addToCart"

        - NOTE: The method "addToCart()" will be implemented next


    - Create a method named "addToCart"
        - Located above the method "render()"

        - NOTE: This serves as the callback method 
            - For the "click" event w/in the Class "ProductItem"


    - Inside the method "addToCart()"
        - Pass the string value "Adding to cart" as argument to "console.log()"

        - Pass the Class property "product" as argument to "console.log()"
            - Prefixed w/ the keyword "this"
            - NOTE: The keyword "this" refers to the Class "ProductItem"


    - Save the chanages
        - Load / reload the browser
        
        - Launch the browser's JavaScript console

        - Click the button "Add to Cart" of a product item

        - On the browser's JavaScript console
            - The message "Adding to cart" is rendered

            - HOWEVER, does NOT render a "ProductItem" object
                - Instead the value "undefined" is returned


    - Explanation of the problem
        - The method "addToCart()" is triggered by the Event handler 
            - Stored w/in the local constant "addCartButton"

        - JavaScript IMPLICITLY BINDS the keyword "this" to the source of an event

        - The source of the "click" event is the Event listener for the
            - Element node "button" stored in the local constant "addCartButton"
            - NOT the Class "ProductItem"
        
        - Thus, JavaScript returns the value "undefined"


    - To address this concern
        - Bind the method "addToCart()" w/ the Class "ProductItem"
        
        - On the Event listener's parameter field
            - Chain the method ".bind()" to the method "addToCart()"
            - Pass "this" as argument to the method ".bind()"

        - Doing so, JavaScript binds the keyword "this" argument to method ".bind()"
            - To what the keyword "this" refers w/in the method "addToCart()"

            - To the same thing to what the keyword "this" refers to 
                - The entire Class "ProductItem"

            - Assuming that the method "render()" is called 
                - On an instance of the Class "ProductItem"


    - Back on the method "render()"
        - Refactor the 2nd argument of the method ".addEventListener()"
            - Chain the method ".bind()" to the method "this.addToCart"
            - Pass the keyword "this" as argument


    - Save the chanages
        - Load / reload the browser

        - Launch the browser's JavaScript console

        - Click the button "Add to Cart" of a product item

        - On the browser's JavaScript console
            - The message "Adding to cart" is rendered
            - Details of the "ProductItem" object is rendered
