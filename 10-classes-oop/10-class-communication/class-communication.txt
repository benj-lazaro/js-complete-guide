Communication Can Be Challenging
--------------------------------
- For this lesson
    - Implement the logic that allows Classes to communicate

    - Calling the callback method "addToCart()" of the Class "ProductItem"
        - Updates the HTML markup w/in the local constant "cartEl"
            - Located w/in the method "render()" of the Class "ShoppingCart"

        - Change the static value of "Total" from 0 to 1


- On the practice project (assets/scripts/app.js)
    - On the Class "ShoppingCart"
        - Create a new method named "addProduct"
        - Define the parameter variable "product"
        - NOTE: This holds the clicked "ProductItem" object added to the cart


    - On the method "addProduct()"
        - Access the Class field "items"
            - Prefixed w/ the keyword "this"

        - Chain the method ".push()"
            - Pass "product" as argument
            - Pushes the "ProductItem" object into the Class field "items"


    - On the method "render()" of the Class "ShoppingCart"
        - Dynamically create a Class property named "outputTotal"
            - Prefixed w/ the keyword "this"

        - Grab & assign as value the Element node "h2" taken from
            - The HTML markup stored w/in the local constant "cartEl"
            
            - Chaining the method ".querySelector()" on the constant "cartEl"
                - Passing "h2" as argument
            
            - Then assign the returned Element node "h2" as initial value
                - To the Class property "outputTotal"

        - NOTE: JavaScript ALLOWS to create a Class property anywhere in
            - Anywhere w/in the Class
            - Does NOT have to be w/in a Constructor method


    - Back to the method "addProduct()"
        - Access the created Class property "outputTotal"
            - Prefixed w/ the keyword "this"

        - Overwrite the (previous) HTML markup w/ an updated markup
            
            `<h2>Total: \$ ${1}</h2>`

        - NOTE: Will implement the logic to dynamically change the value


    - The intention of the implemented changes is 
        - To have the value of "Total" updated
            - EVERYTIME the method "addToCart()" of the Class "ProductItem" is called 

        - For the moment
            - Calling the method "addProduct()" of the Class "ShoppingCart" w/in
                - The Class "ProductItem" will NOT WORK

                - e.g. "ShoppingCart.addProduct()"
            
            - As the Class "ShoppingCart" is NOT instantiated w/in the Class "ProductItem"


    - To address this concern
        - Requires the use of Static methods & properties


    - Save the chanages
        - Load / reload the browser
        - There are NO changes in the behavior of the web app
