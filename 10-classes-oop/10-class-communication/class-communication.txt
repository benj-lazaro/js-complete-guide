Communication Can Be Challenging
--------------------------------
- For this lesson
    - Implement the logic that allows communication between Classes "ShoppingCart" & "ProductItem"

    - The HTML markup stored as value to the Class property "outputTotal" of the Class "ShoppingCart"
        - Will be replaced with an updated HTML markup (i.e. changing the value of "Total" from 0 to 1)
        - EVERYTIME the Class method "addToCart()" of the Class "ProductItem" is called
    

- On the practice project (assets/scripts/app.js)
    - On the Class "ShoppingCart"
        - Create a new Class method named "addProduct"
            - Located above its sibling Class method "render()"
        
        - Define a parameter variable named "product"
            - NOTE: Parameber variable holds the "ProductItem" object added to the cart 
    

    - Inside the Class method "addProduct()"
        - Access the Class field "items" prefixed w/ the keyword "this"
            - Chain the method ".push()"
            - Pass the parameter variable "product" as argument


    - On the Class method "render()"
        - Before the last line of code
            - Returning the local constant "cartEl"

        - Dynamically create a new Class property named "outputTotal"
            - Prefixed w/ the keyword "this"

            - Assign w/ the returned Element node "h2" from the local constant "cartEl"
                - Using the method ".querySelector()"
                - Passing "h2" as argument

            - NOTE: The Class property "outputTotal" now contains the HTML markup & 
                - The "sum" of "ProductItem" objects' prices

        - IMPORANT: JavaScript allows the creation of a Class property 
            - Anywhere w/in a Class
            - It does NOT have to be done inside the Constructor method

    
    - Back on the Class method "addProduct()"
        - Access the Class property "outputTotal" prefixed w/ the keyword "this"

        - Copy the HTML markup of the Element node "h2" that was assigned as value
            - To the ".innerHTML" property of the local constant "cartEl"
            - Located w/in the Class method "render()"
        
        - Paste the copied Element node "h2" as assigned value
            - Overwrites previous HTML markup assigned by the Class method "render()"
            
        - Change the embedded string template literal "${0}" to "${1}"

        - NOTE: The logic for the dynamic update of the string template literal
            - Will be implemented in the succeeeding lessons



    - As mentioned, the intention of the implementation is to 
        - Update the assigned HTML markup in the Class "ShoppingCart" property "outputTotal"
        - EVERYTIME the Class "ProductItem" method "addToCart()" is called


    - Based on the codebase' current state
        - Directly calling the Class "ShoppingCart" method "addProduct()" from inside
            - The Class "ProductItem" will NOT WORK
        
        - Due to the Class "ShoppingCart" NOT being instantiated w/in the Class "ProductItem"
        

    - To address this concern
        - Requires the use of Static methods & properties

        - This allows to call the Class "ShoppingCart" method "addProduct()"
            - Using the following syntax

                ShoppingCart.addProduct(this.product);


    - Save the chanages
        - Load / reload the browser
        - There are NO changes in the behavior of the web app
