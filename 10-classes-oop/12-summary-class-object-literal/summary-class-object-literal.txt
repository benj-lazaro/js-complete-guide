First Summary & Class VS Object Literal
---------------------------------------
- A Class w/ Static methods & properties (fields)
    - Share functionality & data 
        - Accross different parts of the app

    - Serves as a communication interface between classes


- Instantiating a Class into an object
    - Identical to using an Object literal notation
    - It yields a normal object


- When to use a Class
    - Creating the same type of object over & over again
        - Contains the same internal structure but w/ different data

    - Initially have an overhead
        - Offers easy object duplication


- When to use Object literal notation
    - General grouping of (related) data
        - Create an object ONCE
        - NO plan of reusing the object 

    - Offers an easy & quick way to create an object w/ NO overhead
