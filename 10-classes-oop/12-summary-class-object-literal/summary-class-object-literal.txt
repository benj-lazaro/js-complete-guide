First Summary & Class VS Object Literal
---------------------------------------
- A class w/ static methods & properties
    - Share functionality accross different part of the app
    - Share data 
    - Serve as a communication interface between classes


- Instantiating a class into an object is 
    - IDENTICAL to using the object literal notation
    - BOTH yields a normal object 
    

- When to use a Class
    - Create the same type of object over & over again
        - Same internal structure but w/ different data

    - More overhead initially
        - BUT easy "object duplication" thereafter


- When to use the Object Literal notation
    - General data grouping
        - Create an object once
        - NO plans of reusing the object

    - Quick & easy to create
        - NO overhead
