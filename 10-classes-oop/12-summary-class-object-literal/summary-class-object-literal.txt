First Summary & Class VS Object Literal
---------------------------------------
- A Class w/ Static methods & properties (fields)
    - Share data & functionality across different parts of an app
    - Serving as a "glue" (or communication interface) between Classes


- Instantiang a Class to an object is identical to
    - Using the Object literal notation
    - BOTH approaches yield an object 


- When to use a Class
    - Creating an object multiple times over that
        - Contain the same structure (i.e. properties & methods)
        - BUT each instance contain different (unique) data

    - Initially comes w/ an overhead but 
        - Offers an easy way to duplicate objects


- When to use the Object literal notation
    - Creating an object ONLY ONCE
        - To group related data

    - Does NOT come w/ an overhead
        - Offers an easy & quick way to create an object
