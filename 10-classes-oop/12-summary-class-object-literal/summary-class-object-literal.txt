First Summary & Class VS Object Literal
---------------------------------------
- A Class that contain Static methods & properties (fields)
    - Share functionality & data accross different parts of an app

    - To serve as "glue" (i.e. communication interface) between Classes


- Instantiating a Class to an object is
    - Identical to using an Object literal notation

    - BOTH approaches yields an object


- When to use a Class to create an object
    - Create an object multiple times over that
        - Contain the same internal structure (i.e. methods & properties)
        - BUT each instance contains different (unique) data

    - Initially comes w/ an overhead
        - However, offers an easy way to duplicate objects


- When to use Object literation notation to create an object
    - Create an object ONCE that
        - Groups related data


    - Offers an easy & quick way to create an object w/ NO overhead
