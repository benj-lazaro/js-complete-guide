Introducing Inheritance
-----------------------
- The concept of Inheritance
    - To conveniently share code amongst Classes of an app

    - It is implmented by first creating a Base Class that
        - Contain properties & methods that are needed by other Classes

    - Then extend the Base Class into specialized Sub-Class(es)

    - A Sub-Class inherits the properties & methods of the Base Class
        - It can have its own specific properties & methods
        - It can OVERRIDE inherited properties & methods from the Base Class

    - Inheritance is BETTER than creating multiple Classes wherein
        - Each Class contain duplicated properties & methods
        - Doing so is sub-optimal

    - IMPORTANT: A Sub-Class can NOT inherit from multiple Base Classes
        - In JavaScript, a Sub-Class can ONLY inherit from a single Base Class


- On the practice project
    - The Classes "ProductItem", "ProductList" & "ShoppingCart"
        - Noted to have their own Class method "render()" that perform different things
        
        - However, exhibit identical behavior (or general logic)
            - Create an Element node
            - Configure the Element node
            - Return the configured Element node

        - This provide a good opportunity to utilize inheritance
