Introducing Inheritance
-----------------------
- The concept of Inheritance
    - To conveniently share code amongst Classes of an app

    - Create a Base Class that
        - Contains properties & methods needed by other Classes

    - Extend the Base Class into a specialized Sub-Class(es)
    
    - A Sub-Class inherits the Base Class' properties & methods
        - It can have its own specific properties & methods
        - It has the ability to OVERRIDE inherited properties & methods 

    - Inheritance is better than creating multiple Classes wherein
        - Each Class contains code duplication (properties & methods) w/c is sub-optimal


- On the practice project
    - The Classes "ProductItem", "ProductList" & "ShoppingCart" of the web app
        - Have their own method "render()" that perform different things
        - BUT exhibits an identical behavior (i.e. general logic)
            - Create an Element node

            - Configure the Element node
                - Chaining properties such as ".class" & ".innerHTML"

            - Return the new & configured Element node

    
    - This offers a good opportunity to utilize inheritance
