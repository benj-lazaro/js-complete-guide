Implementing Inheritance
------------------------
- For this lesson
    - Implement the concept of inheritance on the practice project

    - Create a Base Class named "Component" that
        - Output different components for the web app

    - Make the current Class "ShoppingCart" into a Sub-Class of "Component"
        - Using the keyword "extends"
        - Followed by the name of the Base Class

    Syntax:

    class <sub-class_name> extends <base_class> {
        ...
    }
    
    - IMPORTANT: JavaScript does NOT support
        - A Class inheriting from multiple Base Classes
        - It can ONLY inherit from a single Class 
    

- On the practice project (assets/scripts/app.js)
    - Create a Base Class named "Component"
        - Located below the Class "Product" but above the rest


    - Inside the Base Class "Component"
        - Create a method named "createRootElement"

        - Define the parameter variables
            - "tag"         = expects a string value
            - "cssClasses"  = expects a string value
            - "attributes"  = expects an array of objects as value


    - Inside the method "createRootElement()"
        - Create an Element node
            - Using the method "document.createElement()"
            - Pass the parameter "tag" as argument
            - Store the created Element node in the local constant "rootElement"

        - Check the value of the parameter variable "cssClasses" 
            - Using an "if" statement & if truthy
                - Chain the property ".className" to the local constant "rootElement"
                - Assign the parameter variable "cssClasses" as value

        - Check the value & length of the parameter variable "attributes"
            - Using an "if" statement & if truthy w/ a length greater than 0
                - Use a "for..of" loop to iterate through the parameter "attributes"

                - Define a local constant "attribute"
                    - To hold the current "name" & "value" pair

                - Chain the method ".setAttribute()" to the local constant "rootElement"
                    - Pass "attribute.name" & "attribute.value" as arguments

            - NOTE: The parameter variable "attributes" will hold an array
                - Containing objects returned by the Class "ElementAttribute"

                - Currently, the Class "ElementAttribute" is not instantiated anywhere
                    - Will do so in the succeeding lessons

                - Commenting the Class "ElementAttribute" does NOT break the app

   
    - Create a Class named "ElementAttribute"
        - Outside of the Base Class "Component"

    
    - Inside the Class "ElementAttribute"
        - Create a Cosntructor method

        - Define two (2) parameter variables
            - "attributeName"
            - "attributeValue"

    - Inside the Constructor method
        - Define a Class property named "name"
            - Prefix w/ the keyword "this"
            - Assign the parameter variable "attributeName" as value

        - Define a Class property named "value"
            - Prefix w/ the keyword "this"
            - Assign the parameter variable "attributeValue" as value
                

    - Back on the Base Class "Component"
        - Create a Constructor method
            - Define a parameter variable named "renderHookId"

        - Inside the Constructor method
            - Define a Class property named "hookId"
                - Prefix w/ the keyword "this"
                - Assign the parameter variable "renderHookId" as value

        - Back on the Class method "createRootElement"
            - At the last line of code 
            
            - Call the method "document.getElementById()"
                - Pass the Class property "this.hookId" as argument
                
            - Chain the method ".append()"
                - Pass the local constant "rootElement" as argument

            - Return the local constant "rootElement"
                - Using a "return" statement


    - Switch to the Class "ShoppingCart"
        - Convert it to a Sub-Class of the Base Class "Component"
            - Using the keyword "extends"
            - Followed by the Base Class "Component"

        - On the Class method "render()"
            - Delete two (2) lines of code
                - Creation of the local constant "cartEl"
                - Chaining of the property ".className" to the local constant "cartEl"

            - Replace it w/ a call to the Class method "createElement()" of
                - The Base Class "Component"

                - Pass the arguments"
                    - 1st   = "section"
                    - 2nd   = "cart"

                - NOTE: Creates an Element node "section"  w/ 
                    - A "class" attribute of "cart"

            - Store the created Element node to the local constant "cartEl"

        - Save the chanages
            - Load / reload the browser

            - Browser reports the error 

                "Uncaught TypeError: Cannot read property "append" of null at ShoppingCart.createRootElement" 

            - The is triggered by the Class field "hookId" of the
                - Constructor method of the Base Class "Component"

                - JavaScript does NOT know the value of the parameter variable "hookId"

            - IMPORTANT: Two (2) rules regarding Constructor method 
                - If a Sub-Class does NOT have a Constructor method
                    - The Base Class' Constructor method will be called

                - If a Sub-Class does have a Constructor method
                    - It uses its own Constructor method
                    - Ignores the Base Class' Constructor method

            - To address this concern
                - Create a Constructor method in the Class "ShoppingCart"
                    - Located below the Getter & Setter pair

                    - Define the parameter variable "renderHookId"

                - Inside the Constructor method
                    - Use the keyword "super()"
                    - Pass the parameter variable "renderHookId" as argument

            - IMPORTANT: Using "super()" w/in the Constructor method of a Sub-Class
                - Ensure that code is NOT DEPENDENT on any Class properties w/in the
                    - Constructor method of the Base Class

                - When defniing Class properties w/in the Constructor method of a Sub-Class
                    - Place them AFTER the keyword "super()"
    
                - Remember, call the Constructor method of the Base Class FIRST
                    - BEFORE defining / using Class properties inside the
                    - Constructor method of the Sub-Class
                    
        - Create a Constructor method
            - Define the parameter variable "renderHookId"

        - Inside the Constructor method
            - Use the keyword "super()"
                - Pass the parameter variable "renderHookId" as argument

            - This effectively calls the Constructor method of the Base Class "Component"
