Implementing Inheritance
------------------------
- For this lesson
    - Implement the concept of inheritance on the Classes of the web app

    - Create a Base Class named "Component"
        - Output an Element node that will be used by a Sub-Class

    - Refactor the Class "ShoppingCart" into a Sub-Class
        - Inherit properties & method of the Base Class "Component"

    Syntax:
    class <name> extends <base_class> {
        ...
    }

    - IMPORTANT: A Sub-Class can NOT inherit from multiple Base Classes
        - In JavaScript, a Sub-Class can ONLY inherit from a single Base Class 
    

- On the practice project (assets/scripts/app.js)
    - Below the Class "Product"
        - Create a Base Class named "Component"


    - Inside the Base Class "Component"
        - Create a Class method named "createRootElement"

        - On the parameter field, define the following parameter variables
            - "tag"         = expects a string value
            - "cssClasses"  = expects a string value
            - "attributes"  = expects an array of objects as value


    - Inside the Base Class "Component" method "createRootElement()"
        - Create an Element node
            - Using the method "document.createElement()"
            - Pass the parameter variable "tag" as argument
            - Store the created Element node to the local constant "rootElement"

        - Check the value in the parameter variable "cssClasses"
            - Using an "if" statement

            - If the condition is truthy
                - Chain the property ".className" to the local constant "rootElement"
                - Assign the parameter variable "cssClasses" as value

        - Check the value in the parameter variable "attributes"
            - Using an "if" statement

            - If the condition is truthy & length (of the array) is greater than 0
                - Iterate through the elements using a "for..of" loop

                - Define the local constant "attribute"
                    - To hold the current pair of "name" & "value" 

                - Inside the "for..of" loop
                    - Chain the method "setAttribute()" to the local constant "rootElement"
                    - Pass "attribute.name" & "attribute.value" as arguments
            
            - NOTE: The parameter variable "attributes"
                - Stores an array of "ElementAttribute" objects

                - An "ElementAttribute" object has two (2) properties
                    - "name"    = HTML attribute name
                    - "value"   = Assigned value


    - Create a Class named "ElementAttribute"
        - Defined BEFORE the Base Class "Component"

        - Create a Constructor method
            - On the parameter field, define the following parameter variables
                - 1st = "attributeName"
                - 2nd = "attributeValue":

        - Inside the Constructor method
            - Define the Class property "name"
                - Prefixed w/ the keyword "this"
                - Assign the parameter variable "attributeName" as value

            - Define the Class property "value"
                - Prefixed w/ the keyword "this"
                - Assign the parameter variable "attributeValue" as value


    - On the Base Class "Component"
        - Create a Constructor method
            - On the parameter field, define the parameter variable "renderHookId"

        - Inside the Constructor method
            - Define the Class property "hookId"
                - Prefixed w/ the keyword "this"
                - Assign the parameter variable "renderHookId" as value

        - On the Class method "createRootElement()"
            - Call the method "document.getElementById()"
                - Pass the Class property "hookId" as argument

            - Chain the method "append()"
                - Pass the local constant "rootElement" as argument

            - Return the local constant "rootElement"
                - Using a "return" statement


    - On the Class "ShoppingCart"
        - Convert it to a Sub-Class of the Base Class "Component"

        - On the Class method "render()"
            - Delete the two (2) lines of code
                - Declaration of the local constant "cartEl"

                - Chaining of the property ".className" to the local constant "cartEl"
                    - Assigned w/ the value of "cart"

            - Leaving the chaining of the property ".innerHTML" 
                - To the local constant "cartEl" behind

            - Replace those lines w/ a call to the 
                - Base Class "Component" method "createRootElement()"
                - Pass the following arguments
                    - 1st = "section"
                    - 2nd = "cart"

            - Store the returned Element node "section" as value
                - To the local constant "cartEl"


    - Save the changes
        - Load / reload the browser
        
        - Browser reports the following error
    
            "Uncaught TypeError: Cannot read property "append" of null at ShoppingCart.createRootElement" 
    
    
    - Root cause
        - Triggered by the Base Class "Component" Constructor method

        - JavaScript does NOT know the current value of the 
            - Constructor method's parametervariable "hookId"

        - IMPORANT: Two (2) rules regarding Constructor method
            - If a Sub-Class does NOT have its own Constructor method
                - The Base Class' Constructor method will be called

            - If a Sub-Class has its its Constructor method
                - JavaScript uses that Constructor method
                - Ignores the Base Class' Constructor method

    - To address this concern
        - Create a Constructor method in the Sub-Class "ShoppingCart"
            - On the parameter field, define the parameter variable "renderHookId"

        - Inside the Constructor method
            - Use the keyword "super()"
            - Pass the parameter variable "renderHookId" as argument

            - NOTE: This calls the Base Class "Component" Constructor method


        - IMPORTANT: When using "super()" w/in a Sub-Class' Constructor method
            - Ensure that the code is NOT DEPENDENT on 
                - Any of the Base Class' properties used w/in its Constructor method

            - Defining or using Sub-Class' properties w/in its own Constructor method
                - MUST be placed AFTER the call to "super()"

            - REMEMBER
                - Call the Base Class' Constructor method FIRST
                - BEFORE defining or using Sub-Class' properties w/in its own Constructor method


    - Back on the Class "ShoppingCart"
        - Right after the Getter "totalAmount()"
            - Create a Constructor method
            - On the parameter field, define the parameter variable "renderHookId"

        - Inside the Constructor method
            - Use the keyword "super()"
            - Pass the parameter variable "renderHookId" as argument 
            

    - On the Class "Shop" method "render()"
        - Refactor the declaration of the local constant "cartEl"
            - Remove the keyword "const" & constant "cartEl"

            - Leaving behind a call to the Sub-Class "ShoppingCart" method "render()"

            - Remove the chained method "append()" from the local constant "renderHook" that
                - Pass the (now removed) local constant "cartEl" as argument


        - On the Class field "cart"
            - Pass "app" as argument to the instantiation of the Sub-Class "ShoppingCart"

            - NOTE: identifies the HTML element by the value of attribute "id" where
                - The Element node "section" will be rendered


    - On the Sub-Class "ShoppingCart" method "render()"
        - Delete the line of code that returns the local constant "cartEl"


    - Save the changes
        - Load / reload the browser
        - The encountered error message has been address
        - The behavior of the web app restored
