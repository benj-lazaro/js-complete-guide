The Prototype Chain & The Global "Object"
-----------------------------------------
- The Prototype Chain does NOT end after reaching out to
    - The "prototype object" assigned to the created object 


- Accessing a property or method that either
    - NOT defined w/in a created object's Constructor function
    - NOT dynamically added to the Constructor function that created the object

    - JavaScript reaches out to the created object property ".__proto__"
        - Access the assigned "prototype object" 

        - The assigned "prototype object" has its own property ".__proto__"
            - Assigned w/ another "prototype object"

        - This "prototype object" comes from 
            - The globally available Constructor function "Object"
            

- The "Object" is a built-in Constructor function that
    - Comes w/ pre-defined properties & methods

    - They become part of the "prototype object" of the
        - Default "prototype object" of the created object

    - The "fallback object" of ALL objects is the "Object.prototype" &
        - NOT the "Object" itself

        - Assigned w/ a "prototype object" whose property ".__proto__"
            - Does NOT have an assigned "prototype object"
            - Accessing it returns a value of "null"

            - NOTE: This is the end of the Prototype Chain
            
    - NOTE: The property ".prototype" exists on a Constructor function
        - The "Object" is a Constructor function 

        - The property ".prototype" defines what will be assigned
            - As a "prototype object" to ANY object that is
            - Created based on this Constructor function


- Every created JavaScript object
    - Whether using a Class, Constructor function or Object literal notation
        - By default, uses the Global Constructor function "Object"

    - JavaScript use the "Object.prototype" as a "fallback object" in addition
        - To the default "prototype object" assigned by a Constructor function
