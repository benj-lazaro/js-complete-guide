Classes & Prototypes
--------------------
- The Constructor function property ".prototype"
    - Configures what will be added to the "prototype object" that
        - Will be assigned to the property ".__proto__" of the created object
        - Based on that Constructor function

    - EVERY JavaScript object has a property ".__proto__" points to
        - The "prototype object" that has been assigned to the object
        - Serves as a "fallback object"
    
    - Important to understand the difference
        - As this is the core idea behind prototypes


- Using Classes & the keyword "extend" for inheritance
    - Makes handling "prototype object" much easier

    - It is super important to understand what happens behind-the-scenes
        - JavaScript is a prototype-based language

    - Useful to special case code implementations where Classes are NOT allowed
    

- A Constructor is NOT a perfect replacement for a Class
    - Code inside a Class Constructor method
        - Can go directly into a Constructor function
        - Except for the call to a Base Class Constructor method using "super()"
        - Performs like a Class would

    - Class fields are translated as properties in a Constructor function

    - Class methods are treated differently when compared to a Constructor function


- On the example code (app.js)
    - The Base Class "AgedPerson"
        - Contains a Class method named "printAge"
        - It access the created object's instance property ".age"
        - Pass it as an argument value to "console.log()"


    - The Sub-Class "Person"
        - Inherits from the Base Class "AgedPerson" using the keyword "extends"

        - Contains 
            - Two Class fields "name" & "age"

            - A Class method named "greet()" that 
                - Accesses the Class field "age"
                - Embedded it w/ in a string literal 
                - Passed as argument value to "console.log()"

    
    - Sub-Class "Person" instantiated into a Class-object
        - Stored in a global constant named "p"

        - Pass the global constant "p" as argument to "console.log()"
            - To view the "prototype object" assigned as value to the property ".__proto__"


    - The "prototype object" assigned to the "p" object
        - Points to the Sub-Class "Person"
            - Evident on the "prototype object" property ".constructor"
            - Although the "prototype object" is labeled as "AgedPerson"

        - The assigned "prototype object" has its own property ".__proto__" that
            - Points to the Base Class "AgedPerson"
                - Evident on the "prototype object" property ".constructor"

        - JavaScript have added an extra "prototype object"
            - When working w/ Classes

    
    - As for the Sub-Class "Person" fields "name" & "age"
        - They are directly added as properties to the created object "p"

        - These Class fields are added to the (to be) created object
            - Right after a call to te Base Class using "super()"
            - Inside the Sub-Class "Person" Constructor method
