Classes & Prototypes
--------------------
- The Constructor function property ".prototype"
    - Configures what will be added into the default "prototype object" of 
        - Objects that are created based on that Constructor function
    
    - The property ".__proto__" w/c is available on EVERY JavaScript object
        - Points to the "prototype object" that has been assigned to an object
        
    - NOTE: It is important to understand the difference between the two
        - This is the idea behind prototypes


- Using Classes & the keyword "extend"
    - Makes prototypical inheritance easier

    - It is important to understand what happens behind-the-scenes (i.e. prototype)
        - As JavaScript is based on it
            - A prototype-based language that use prototypical inheritance

        - It is a core concept in JavaScript that has been around forever


- A Constructor function is NOT a perfect replacement for a Class
    - A Class Constructor method code is implemented directly on a Constructor function
        - Except for the call to a Base Class Constructor method using "super()"    
        - Performs exactly like a Class would

    - A Class field is translated as a property in a Constructor function
    
    - The difference between a Class & a Constructor function 
        - Lies on how a method is added


- On the example code (app.js)
    - The Base Class "AgedPerson" has a method "printAge()" that
        - Access an object's instance property ".age"
        - Pass it as an argument value to "console.log()"


    - The Sub-Class "Person"
        - Use the keyword "extends" to inherit from the Base Class "AgedPerson"
        - Defined w/ properties named "name" & "age"
        - Defined w/ a method named "greet()"


    - Instantiate the Sub-Class "Person" using the keyword "new"
        - Stored the created object in the global constant "p"

        - Pass the constant "p" as argument to "console.log()"
            - To view the assigned "prototype object" to the property ".__proto__"

    
    - The "prototype object" assigned on the created "Person" object property ".__proto__"
        - Points to the Sub-Class "Person"
            - Evident on the property ".constructor"
            - Despite the "prototype object" labeled as "AgedPerson"

        - The "prototype object" has its own property ".__proto__" that
            - Points to the Base Class "AgedPerson"

        - JavaScript added an extra "prototype object" when working w/ Classes

    
    - The Sub-Class Person properties "name" & "age" where
        - Directly added as properties of the created object

        - The reason behind is that
            - The Sub-Class properties are added ONLY into the (to be) created object
            - After a call to the Base Class using "super()" inside a Constructor method

        - Methods however are treated differently
