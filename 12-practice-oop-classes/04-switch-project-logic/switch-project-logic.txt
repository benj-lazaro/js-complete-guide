- On the practice project (assets/scripts/app.js)
    - On the Class "ProjectList"
        - Define a new Class field named "projects"
        - Assign an empty array


    - On the Class "ProjectList" Constructor method
        - Use a "for..of" loop
            - To iterate through the NodeList in the local constant "projectItems"

        - For each Element node "li" that is read from the NodeList
            - Store each Element node "li" read in the local constant "projectItem"

        - Inside the "for..of" loop
            - Access the Class field "projects" (as a Class property)
                - Chain the method ".push()"
    
            - Create an instance of the Class "ProjectItem"

                - (e.g. "projectItem.id")

            - Pass the created "ProjectItem" object as argument value to the method ".push()"
        

    - On the Class "ProjectItem"
        - Create a Constructor method w/ a parameter variable named "id"

        - Create two (2) Class methods named
            - "connectSwitchButton"
            - "moreInfoButton"


    - On the Class "ProjectItem" Constructor method
        - Create a Class property named "id"
            - Assign the parameter variable "id" as value

        - Add the created Class methods "connectSwitchButton()" & "moreInfoButton()"
            - Prefixed w/ "this"


    - On the Class "ProjectItem" method "connectSwitchButton()"
        - Select the HTML element <li> by its attribute "id"
            - Using the method "document.getElementById()"
                - Pass the Class property "this.id" as argument value

            - Store the selected HTML element <li"> 
                - To the local constant "projectItemElement"

        - Select the last HTML element <button> of the fetched HTML element <li>
            - Chaining the method ".querySelector()" to the local constant "projectItemElement"
                - Passing "button:last-of-type" as argument value

            - Store the selected HTML element <button>
                - To the local constant "switchBtn"

        - Add a listener for an Event "click" to the local constant "switchBtn"
            - Chaining the methd ".addEventListener()"
            - Passing the event "click" as the 1st argument value
            
            - NOTE: The callback function to be implementated on the next lessson
                - This is supposed to be part of the Class "ProjectList"

            - The switch between "active-projects" & "finished-projects" is
                - To be performed by the Class method
                    
            - The Class "ProjectList" handles project lists that are
                - Categorized as "active-projects" & "finished-projects"

            - Clicking the button "Finish" of an project item under "Active Projects"
                - Will move the project item to "Finished Projects"
                - And vice-versa for a project item under "Finished Projects"

            - To address this concern
                - A callback function should be attached & managed by the Class "ProjectList"


    - On the Class "ProjectList"
        - Create two (2) Class methods
            - "addProject()"
            - "switchProject()"


    - On the Class "ProjectList" method "switchProject()"
        - Define the parameter variable "projectId"

        - Solution #1 (longer implementation)
            - Identify the index position of the "ProjectItem" object
                - Whose button "Finish" was clicked
                - From the Class field "projects"

            - Chain the method ".findIndex()" to the Class field "projects"
                - Pass an Arrwow function as argument value

            - The Arrow function has a parameter variable named "p"
                - The parameter variable "p" holds the HTML element <li> read

                - Compare the property "id" of the HTML element <li> w/ 
                    - The parameter variable "projectId"

                - If it is STRICTLY EQUAL
                    - Assign the returned index value of HTML element <li> 
                    - To the local constant "projectIndex"

            - Access the Class field "projects" (as a property)
                - Chain the method ".splice()"
                - Pass the argument values
                    - 1st = local constant "projectIndex"
                    - 2nd = 1

                - Removes & returns the HTML element <li> from the Claas field "projects"

        - Solution #2: (shorter implementation)
            - Access the Class field "projects" (as a property)
                - Chain the method ".filter()"
                    - Pass an Arrow function as argument value

            - The Arrow function has a parameter variable named "p"
                - The parameter variable "p" holds the HTML element <li> read

                - Compare the property "id" of the HTML element <li> w/ 
                    - The parameter variable "projectId"

                - If it is NOT STRICTLY EQUAL
                    - Assign the returned index value of HTML element <li> 
                    - To the local constant "projectIndex"

            - Assign the new array (overwriting) back to the Class field "projects"
