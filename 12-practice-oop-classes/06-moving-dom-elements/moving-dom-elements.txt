Moving DOM Elements
-------------------
- Implement the logic on the Class "ProjectList" method "addProject()"
    - Moves a "ProjecItem" object's DOM element to
        - The neighboring DOM element of a "ProjectList" instance 


- On the practice project (assets/scripts/app.js)
    - On the Class "ProjecList" method "addProject()"
        - Add a parameter variable named "project"

        - NOTE: Holds a "ProjectItem" object that is currently read
                    

    - Inside the Class "ProjecList" method "addProject()"
        - Access the Class "ProjecList" field "projects"
            - Chain the method ".push()"
            - Pass the parameter variable "project" as argument value

        - NOTE: Pushes the "ProjecItem" object read to the Class field "projects" of
            - The other (neighboring) Class "ProjectList" instanced
            - Stored in the local constants "finishedProjectsList" or "activeProjectsList"
            - Located w/in the Class "App"


    - Create a new Class named "DOMHelper"
        - This Class will be responsible for moving the Class "ProjecItem" object's
            - DOM elements to the other Class "ProjecList" instance DOM element

        - Create a Static method named "moveElement"
            - Define two (2) parameter variables
                - 1st = "elementId"
                - 2nd = "newDestinationSelector"


    - On the Class "DOMHelper" Static method "moveElement()"
        - Create a local constant named "element"
            - Assign the method "document.getElementById()" as value
            - Pass the parameter variable "elementId" as argument value

            - NOTE: Fetch & store the corresponding Element node "li" of 
                - A "ProjecItem" object identified by its property "id"

        - Create another local constant named "destinationElement"
            - Assign the method "document.querySelector()" as value
            - Pass the parameter variable "newDestinationSelector" as argument value

            - NOTE: Fetch & store the Element node "ul" where
                - Element node "li" of a "ProjectItem" object will switch to

        - Acecss the local constant "destinationElement"
            - Chain the method ".append()"
            - Pass the local constant "element" as argument value

            - NOTE: Moves the Element node "li" of a "ProjectItem" object
                - To the new (neighbor) Element node "ul" 
                - Identified by the value of its HTML attribute "id"
                - (i.e. "active-projects" or "finished-projects")


    - On the Class "ProjectList" method "addProject()"
        - Call the Class "DOMHelper" Static method "moveElement()"
            - Pass the following argument values
                - 1st = "project.id"
                - 2nd = `#${this.type}-projects ul`

        - NOTE: Moves the "ProjectItem" object's Element node "li" to the
            - New Element node "ul" identified by its HTML attribute "id"


    - On the Class "ProjectItem" method "connectSwitchButton()"
        - Refactor the callback function attached of the local constant "switchBtn"
            - On the call to the Class property "updateProjectListsHandler"
            - Chain the method ".bind()"
            - Pass the following argument values
                - 1st = null
                - 2nd = "this.id"

        - NOTE: Attaches the "ProjecItem" object's property "id" to the callback function
            - Stored in the Class property "updateProjectListsHandler"


    - On the Class "ProjectList" method "addProject()"
        - Access the parameter variable "project"
            - Chain the Class "ProjectItem" method ".update()"
            - Pass the following arugment values
                - 1st = "this.switchProject.bind(this)"
                - 2nd = "this.type"

        - NOTE: The Class "ProjectItem" method "update" will be implemented next

    
    - On the Class "ProjecItem"
        - Create a new Class method named "update"
            - Define two (2) parameter variables
                - 1st = "updateProjectListsFn"
                - 2nd = "type"

            - NOTE: The property "type" identifies if a "ProjectItem" object
                - Belongs to "Active" or "Finished" Projects

        - NOTE: Updates the switch handler of a "ProjectItem" object


    - On the Class "ProjecItem" method "update()"
        - Create a Class property named "updateProjectListsHandler"
            - Assign the parameter variable "updateProjectListsFn" as value

        - Call the Class method "connectSwitchButton()"
            - Pass the parameter variable "type" as argument value


    - On the Class "DOMHelper"
        - Create another Static method named "clearEventListeners"
            - Define a parameter variable named "element"

        - NOTE: Responsible for clearing previously assigned
            - Event listener to a "ProjectItem" object


    - On the Class "DOMHelper" Static method "clearEventListeners()"
        - Create a local constant named "clonedElement"
            - Assign the parameter variable "element" as value
            - Chain the method ".cloneNode()"
            - Pass the boolean value "true" as argument value

            - NOTE: Creates a deep clone of a "ProjecItem" object's DOM elements

        - Access the parameter variable "element"
            - Chain the method ".replaceWith()"
            - Pass the local constant "clonedElement" as argument value

            - NOTE: Replaces the original "ProjecItem" object's DOM elements w/
                - A brand-new clone of itself & effectively ditches previously
                - Attached Event Listener on the original DOM elements

        - Return the local constant "clonedElement"


    - On the Class "ProjectItem" method "connectSwitchButton()"
        - Add a new parameter variable named "type"

        - Refactor the local constant "switchBtn" into a variable
            - Changing the reserved keyword from "const" to "let"

        - Call the Class "DOMHelper" Static method "clearEventListeners()"
            - Pass the local variable "switchBtn" as argument value
            
            - Store the returned cloned "ProjectItem" object's DOM elements
                - Back to the local variable "switchBtn"

            - NOTE: Effectively replacing the previous one

        - Access the local variable "switchBtn"
            - Chain the property ".textContent"
                - Assign a ternary expression as value

            - The ternary expression
                - Checks if the parameter variable "type" is STRICTLY EQUAL
                    - To the word "active"

                - If true
                    - Assign the value "Finished"

                - Otherwise
                    - Assign the value "Activate"

            - NOTE: This updates the "ProjectItem" object's button text
                - From "Finisned" to "Activate" & vice-versa


    - On the Class "ProjecItem" Constructor method
        - Add a 3rd parameter variable named "type"

        - On the declaration of the Class method "connectSwitchButton()"
            - Pass the parameter variable "type" as argument value


    - On the Class "ProjectList" Consutructor method
        - On the instantiation of the Class "ProjecItem"
            - Pass the Class properyt "type" as the 3rd argument value


    - Save the changes
        - Load / reload the browser

        - Clicking a "ProjectItem" objects button (i.e. Finish or Activate)
            - Moves the "ProjecItem" object to 
                - The corresponding project list (Active or Finished)
                - Without generating errors on the brower's JavaScript console

            - Updates the text content of the "ProjecItem" button accordingly
