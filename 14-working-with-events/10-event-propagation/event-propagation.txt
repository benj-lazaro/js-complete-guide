Event Propagation & "stopPropagation()"
---------------------------------------
- In the browser-side JavaScript
    - By default, an Event listener is registered in the "Bubbling" phase

    - When an Event listener is chained to a selected Element node
        - JavaScript executes it 1st
            - The "Capturing" phase that runs 1st is ignored

        - JavaScript then checks the ancestors for Event listeners
            - If there is one
            - JavaScript executes it

    - This default behavior can be switched
        - By passing "true" as the 3rd argument value
            - To the method ".addEventListener()"
            - By default, the value of "false" is IMPLICITLY passed

        - JavaScript executes the ancestor's Event listener 1st
            - Before the selected Element node's Event listener

            - Effectively executing the "Capturing" phase 1st
            - Then the "Bubbling" phase


- Propagation refers to an Event that
    - NOT only triggers the Event listener of the selected Element node
        - But also of ALL of its ancestors w/ an Event listener
        - The trigger "bubbles" up from the selected Element node to its ancestors


- The "Event" method ".stopPropagation()"
    - Prevents the registered Event listener of the selected Element node's ancestor
        - From being triggered alonside w/ the Element node's Event listener

    - NOTE: Does NOT PREVENT selected Element node's default behavior from executing
        - It still allows for the default behavior ro run

    - IMPORTANT: ONLY prevents the ancestor's Event listener but does NOT prevent
        - Multiple Event listeners registered on the same Element node from being triggered


- The "Event" method ".stopImmediatePropagation)"
    - Useful if a selected Element node has multiple registered Event listeners

    - It immediately stops other registered Event listener from execution
        - After it executed the current one


- To determine of an Event object will propagate or NOT
    - Pass the Event object as argument value to "console.log()"

    - Check the property "bubbles"
        - If "false" the Event object will NOT propagate
        - Otherwise, it will & may require to use the method ".stopPropagation()"
