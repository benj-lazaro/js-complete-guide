More On Regular Expressions
---------------------------
- Regular expression is case-sensitive
    - To create a string pattern that is case-insensitive
        - Wrap the specific pattern w/in a pair of "( )"
        - Inside place both lowercase & uppercase pattern, separated by a "|"


- To implement a wildcard character
    - Replace the target character of the string pattern w/ a "."
        
    - Allows to accepts ANY character in-place
        - Evaluates the string pattern to be "true"

    - Absense of a character in-place
        - Evaluates the string pattern to be "false"


- To escape characters
    - Prefix a backslash character on the character to be escaped
        - e.g. \. = to treat "." literally as it is & NOT as a wildcard character
        - e.g. \S = to search for any word & NOT a literal uppercase "S" character


- The method ".exec()"
    - Chained to a regular expression
        - Pass string pattern as an argument value
        - Returns an array w/ some information about the string
    
    - The "index" identifies where exactly in the string the regular expression pattern matches


- The method ".match()"
    - Chained to a string
        - Pass regular expression as an argument value
        - Returns an array w/ some information about the string

    - The "index" identifies where exactly in the string the regular expression pattern matches


- IMPORTANT: Learning & writing complex regular expressions takes time to master
    - It is not needed that often
    - Best to lookup online for the best pattern for specific use-cases
    - Particularly when doubtful the stirng pattern created is the best one
