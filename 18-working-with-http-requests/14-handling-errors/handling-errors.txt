Handling Errors
---------------
- Implement logic that handles errors brought about by a failed HTTP request


- The "XMLHttpRequest" object property ".onerror"
    - Handles network-related errors that prevent a HTTP request from being sent

    - Pass a new "Error" object w/ a customized message as argument value to the function "reject()"


- The "XMLHttpRequest" object property ".status"
    - Handles errors where the HTTP request is sent & received a HTTP response containing an error
        - Called w/in the property ".onload"

        - Use an "if" statement to check the HTTP status code
            - Stored w/in the returned HTTP response property ".status"
    
        - HTTP status code between 200 & 300 meant
            - Pass the returned "XMLHttpRequest" object as argument value to the function "resolve()"

        - Otherwise
            - Pass a new "Error" object w/ a customized message as argument value
                - To the function "reject()"
